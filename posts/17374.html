1.什么是JVM？

定义：

​ Java Virtual Machine - Java 程序的运行环境(Java 二进制字节码的运行环境)

好处：

  • 一次编写，到处运行
  • 自动内存管理 垃圾回收功能
  • 数组下标越界检查
  • 多态

比较：

​ jvm jre jdk

jdk：JVM+基础类库+编译工具

jre：JVM+基础类库

jvm：运行环境

2.学习JVM有什么用？

  • 面试
  • 理解底层的实现原理
  • 中高级程序员的必备技能

3.常见的JVM

1.程序计数器

1.1 定义

Program Counter Register 程序计数器(寄存器)

  • 作用，是记住下一条jvm指令的执行地址
  • 特点
    • 是线程私有的(每个线程都有他们自己的程序计数器)
    • 不会存在内存溢出(程序计数器唯一 一个不存在内存溢出)

1.2作用

右侧是Java 源代码 这些源代码我们都知道不能直接被执行 他得经过一次编译 编译成我们左侧的二进制字节码 左侧好像看不懂… 没关系他其实是jvm指令

那这些指令能够直接交给cpu来执行吗？

还不行 必须要经过一个叫解释器才能执行

如下:

​ 二进制字节码 ===>解释器 ===>机器码 ===> CPU

将二进制字节码文件交给解释器 让解释器解释成机器码 再交由给CPU执行

那说这么多和我们程序计数器有什么关系呢？

我们说到的程序计数器 他的作用就是在指令执行过程中记住下一条jvm指令的执行地址

根据这个地址信息程序计数器就可以找到他 来执行他

二进制字节码 ===>解释器 ===> 程序计数器 ===>机器码 ===>CPU

在物理上 实现程序计数器是通过一个叫寄存器来实现的 寄存器是整个cpu组件里读取速度最快的一个单元 因为我们读写指令这个动作是非常频繁的 Java在设计的时候就把我们cpu中的寄存器当作了我们的程序计数器 用它来存储地址 将来去读取这个地址

2.虚拟机栈

栈数据结构就好像子弹夹

当你需要使用他的时候他就会找到你最上方的子弹让他出栈

以此类推

先进后出这种原理

那Java中的虚拟机栈他到底是干什么用的呢 其实大家想一想 我们Java中每一个线程

运行的时候需不需要给每个线程划分一个内存空间呢 ？ 答案是肯定的

一个线程运行它需要一个栈 如果将来它有多个线程它就会有多个虚拟机栈 那每个栈内它又由什么组成的呢 ？

一个栈内由多个栈帧组成

那栈帧又是什么呢？

一个栈帧就对应着一次方法的调用 线程最终是去执行代码的 而代码都是由一个个方法组成

所以在线程运行的时候 每个方法所需要的内存我们就称之为栈帧

栈帧 - 每个方法运行时所需要的内存

那我方法需要什么内存呢？

参数/局部变量/有返回值的话还需要返回地址…

方法执行就等同于方法入栈 方法执行完毕等同于方法出栈 也就将内存释放掉了

一个栈由多个栈帧组成

2.1 定义

Java Virtual Machine Stacks(Java 虚拟机栈)

  • 每个线程运行时所需要的内存 称为虚拟机栈
  • 每个栈由多个栈帧(Frame)组成 对应着每次方法调用时所占用的内存
  • 每个线程只能有一个活动栈帧 对应着当前正在执行的那个方法(就是栈顶部的方法就是活动栈帧)

问题辨析

  1. 垃圾回收是否涉及栈内存？

    不需要 因为我们栈内存他无非就是一次次方法的调用所产生的栈帧内存 而栈帧内存呢 在每一次方法调用后都会弹出栈 也就是说会被自动回收掉 所以根本不需要垃圾回收来管理我们栈内存 垃圾回收主要管理堆内存 栈内存不需要让垃圾回收做处理

  2. 栈内存分配越大越好吗？

​ 并不是 栈内存划分的越大 反而会让你的线程数变少 思考一下 因为我们物理内存是一定的 比如说我一个线程他使用的是栈内存 一个线程他假设使用了1兆内存 物理内存有500兆 理论上可以有500个线程同时运行 但是如果我给每个线程栈内存设置为2兆内存 那么理论上你最多同时运行250个线程 所以说并不是划分的越大越好 划分的大了 通常只是能够进行更多次的方法递归调用 而不会增强你运行的效率 反而会影响到线程数目的变少 不建议设置的大一般采用系统默认大小

​ 3.方法区的局部变量是否线程安全？

​ 看一个变量线程是否是安全的 首先要看 这个变量是否是线程共享的还是对每个线程是私有的

  • 如果方法内局部变量没有逃离方法的作用范围 他是线程安全的 反之
  • 如果这个局部变量引用了对象 并逃离方法的作用范围 需要考虑线程安全

3. 本地方法栈

Native Method Stacks(本地方法栈)

有些功能Java有时候不能直接和操作系统打交道 需要借助c/c++编写的本地方法来真正与操作系统底层打交道 而Java代码可以间接的调用本地方法来调用底层的一些功能

这些本地方法使用的内存就是本地方法栈

5. 方法区

5.1定义

所谓的方法区是所有Java虚拟机线程共享的 他存储了跟类的结构相关的信息 field (成员变量)

method data(方法数据) 以及 code for method and constructors(成员方法以及构造器方法代码部分) special method (特殊方法主要是指类的构造器) 以及 run-time constant poo(运行时常量池)

从上方可以了解到 方法区主要存的就是有关于类的信息

方法区会在启动时被创建 他逻辑上是堆的一个组成部分

5.2组成

1.6：

​ 有堆以及Method Area MethodArea只是一个规范而已

​ Method Area 中使用的是一个永久代的作为方法区的实现 这个永久代里包含了几部分内容 它可以存储我们类里面的信息 当然累加器也在里面 还有一块叫运行时常量池

1.8：

​ 永久代这个实现被废弃掉了 他的实现叫元空间 元空间有类的信息 类加载器 常量池 不过呢他已经不再占用堆内存了 换句话说 他已经不是由我们jvm来管理他了 移出到本地内存 所谓的本地内存就是操作系统的内存 他里面还会跑一些其他进程 其中就包含元空间 而StringTable被移植到了堆内存中

5.3 方法区内存溢出

  • 1.8以前会导致永久代内存溢出
  • 1.8之后会导致元空间内存溢出

5.4 运行时常量池

  • 常量池 就是一张常量表 虚拟机指令根据这张常量表找到要执行的类名 方法名 参数类型 字面量等信息
  • 运行时常量池 常量池是 *.class文件中的 当该类被加载 他的常量池信息就会被放入运行时常量池 并把里面的符号地址变为真实地址

二进制字节码(类基本信息 常量池 类方法定义 包含了虚拟机指令)

而我们写的一行行代码会被解析成一个个的指令 指令会去常量池中找对应的StringTable

5.8 StringTable 垃圾回收

StringTable也是会受到垃圾回收的管理 当内存空间不足时 那些没有引用的字符串常量仍然会被垃圾回收掉 很多人错误的认为 字符串常量它就是永久的了 不会被垃圾回收 事实上可以被垃圾回收

5.9 StringTable 性能调优

  • 调整 -XX:StringTableSize=桶个数
  • 考虑将字符串对象是否入池

为什么我们要使用StringTable呢？ 那我们什么情况下要使用到StringTable呢？

先讲网上流传的一个案列 ：

​ 国外有一个互联网公司 是一个社交平台 这个社交平台呢 它要存储大量的用户的信息 包括用户的一些地址信息 因为这个用户量比较大 但这个用户的地址呢都是字符串表示的 我需要用30G的内存才能存的下这么多的信息 但这些用户的地址很多可能是重复的 比如说有一个用户它是湖南长沙 还有可能1000个用户也可能是这个地址 那如果我们不加这个区分 把这些重复的地址都存入内存 那么它占用的内存上限就是30G

解决办法：采用字符串的inter方法入池 这样可以去除重复的地址 这样可以减少字符串对内存的占用 据说使用inter’方法之后 内存下降到了数百兆 就不是30G了 只是个案例
