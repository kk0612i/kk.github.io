JMeter压力测试工具安装以及快速使用

发表于2022-06-24|更新于2022-06-24
|阅读量:

1.JMeter是什么？

​ Apache JMeter 是 Apache 组织基于 Java 开发的压力测试工具，用于对软件做压力测试

​ JMeter最初被设计用于Web应用测试 但后来被扩展到了其他测试领域 可用于测试静态和动态资源 如静态文件 Java小服务程序 GGI脚本 Java对象 数据库和FTP服务器等…. JMeter可对服务器 网络或对象模拟巨大的负载 在不同压力类别下测试他们的强度和分析整体性能

2.JMeter的作用

  1. 能够对Http和FTP服务器进行压力和性能测试 也可以对如何数据库 进行同样的测试(通过JDBC)
  2. 完全的可移植性和100% 纯Java
  3. 完全多线程 框架允许通过多个并发取样和 通过单独的线程对不同的功能同时取样(这个好 可以用来模拟高并发)

3.术语

  1. Lable：定义的Http请求名称
  2. Samples：表示这次测试中一共发出了多少个请求
  3. Average：访问页面的平均响应时间
  4. Min：访问页面的最小响应时间
  5. Max：访问页面的最大响应时间
  6. Error%：错误的请求的数量/请求的总数
  7. Throughput：每秒完成的请求数
  8. KB/Sec：每秒从服务器端接收到的数据量
  9. QPS：QPS即每秒查询率，是对一个特定的查询服务器在规定时间内所处理流量多少的衡量标准
    1. 计算关系：
      1. QPS = 并发量 / 平均响应时间
      2. 并发量 = QPS * 平均响应时间
    2. 原理：每天80%的访问集中在20%的时间里，这20%时间叫做峰值时间
    3. 公式：( 总PV数 * 80% ) / ( 每天秒数 * 20% ) = 峰值时间每秒请求数(QPS)
